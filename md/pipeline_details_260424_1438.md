### 단계별 상세 설명

| 단계 | 파일 | 설명 |
|------|------|------|
| STEP 1: 증분 체크 | `incremental_runner.py` | 파일의 SHA-256 해시를 `registry.json`과 비교하여 변경된 파일만 처리 |
| STEP 2: 포맷 변환 | `doc_ingest.py::to_pages()` | PDF는 PyMuPDF 직접 파싱, Office/HWP는 LibreOffice headless로 PDF 변환, 텍스트 파일은 가상 페이지 생성 |
| STEP 3: 페이지 렌더링 | `doc_page_render.py::render_pdf()` | PDF 각 페이지를 고해상도 JPEG로 래스터화하여 저장 |
| STEP 4: 캡셔닝 | `incremental_runner.py::_caption_for_im()` | Qwen2-VL 비전-언어 모델로 한국어 페이지 설명 생성 (캐시 우선 사용) |
| STEP 5: 3축 임베딩 | `siglip2_re.py`, `bgem3_caption_im.py`, `dinov2_z.py` | 각 페이지를 Re(시각의미)/Im(언어의미)/Z(구조) 3개 벡터로 변환 |
| STEP 6: Sparse/ASF | `bgem3_sparse.py`, `asf_filter.py` | 어휘 기반 희소 표현과 적응형 시맨틱 필터 생성 |
| STEP 7: 캐시 저장 | `cache.py`, `registry.py` | .npy 파일에 벡터 누적, registry 업데이트 |

### Doc vs Img 차이점

| 구분 | Doc | Img |
|------|-----|-----|
| 전처리 | 포맷 변환 → 페이지 렌더링 필요 | 직접 입력 (추가 변환 없음) |
| 세그먼트 단위 | 페이지 (p001, p002, ...) | 파일 1개 = 벡터 1개 |
| 캐시 디렉토리 | `TRICHEF_DOC_CACHE/` | `TRICHEF_IMG_CACHE/` |
| LibreOffice 의존 | 필요 (Office/HWP 변환) | 불필요 |

### 단계별 상세 설명

| 단계 | 파일 | 설명 |
|------|------|------|
| STEP 1: 증분 체크 | `registry.py` | SHA-256으로 변경 파일만 처리 |
| STEP 2: 메타데이터 추출 | `frame_sampler.py::probe_duration()` | FFprobe로 FPS, 총 프레임, 재생시간 파악 |
| STEP 3A: 프레임 추출 | `frame_sampler.py::extract_frames()` | 30프레임 간격으로 대표 프레임 샘플링 |
| STEP 3B: 오디오 추출 | `frame_sampler.py::extract_audio()` | 영상에서 오디오 트랙을 WAV로 분리 |
| STEP 4A: 시각 인코딩 | `vision.py::SigLIP2Encoder` + `DINOv2Encoder` | 프레임 → Re(의미)/Z(구조) 벡터 |
| STEP 4B: STT | `stt.py::WhisperSTT` | Whisper medium으로 한국어 음성→텍스트 |
| STEP 5: 시간 정렬 | `movie_runner.py::_align_stt_to_frames()` | 각 프레임 타임스탬프에 맞는 STT 텍스트 매핑 |
| STEP 6: 텍스트 임베딩 | `text.py::BGEM3Encoder` | STT 텍스트 → Im축 벡터 (1024d) |
| STEP 7: 캐시 저장 | `cache.py`, `registry.py` | .npy 누적, segments.json 저장 |

## 5. 핵심 설계 원리

> **모든 모달리티를 동일한 3축 복소 벡터 공간으로 투영하여 Hermitian 유사도 하나로 통합 검색을 수행한다.**

- **Re축 (실수부)**: 시각적 의미 — SigLIP2가 이미지/프레임을 언어와 정렬된 공간으로 인코딩
- **Im축 (허수부)**: 언어적 의미 — BGE-M3가 캡션/STT 텍스트를 다국어 공간으로 인코딩
- **Z축 (구조부)**: 시각적 구조 — DINOv2가 이미지의 레이아웃/패턴을 인코딩

Music은 시각 정보가 없으므로 Re/Z축이 0벡터이며, Im축(텍스트 의미)만 실질적으로 동작한다.
이를 통해 텍스트 쿼리로 영상의 특정 장면, 문서의 특정 페이지, 음악의 특정 구간을 동시에 검색할 수 있다.
